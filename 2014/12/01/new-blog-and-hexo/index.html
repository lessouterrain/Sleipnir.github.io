Hexo 和新Blog

这篇简单讲讲我在Github上折腾hexo的过程吧。

作为一个伪Geek，大概唯一像Geek的一点就是爱折腾了。折腾这个其实最开始只是因为在Tumblr上好不容易找到一个好看的Theme结果它卖19刀……
于是不幸福的我就到处google，结果就看到Octopress和Hexo了。

当然了，因为我完全没有任何程序相关知识，所以就还是决定善待自己选择了相对简单的Hexo。至于Github，说了我是伪Geek所以还是早有耳闻，但是因为1.用不上 2.不会弄，所以也是从来没用过（跪。
然后随便搜了一个看上去比较详细的教程就开始折腾了。

个人觉得，只要不是只会点鼠标聊QQ看视频的，稍微有些计算机知识基础的人，基本都能搞定，所以也完全不用觉得太复杂弄不明白，何况不是还有那么多前人的教程么（笑。
网上教程太多我个啥都不懂的就不装x再赘述了。
我参考的是Zippera桑的系列教程。算是比较详尽明了的版本。
hexo系列教程：（一）hexo介绍
按着他的教程来就好了。
这里我要补充的一点是，他写的版本时间稍早，当你完成hexo的安装步骤后，请再多打三行命令装三个必要部件，不然传上Github的页面显示出来就只是几行代码了。

$ npm install hexo-renderer-ejs --save
$ npm install hexo-renderer-stylus --save
$ npm install hexo-renderer-marked --save

然后当进行到对主题config的时候，你现在下载下来的Hexo的默认主题不是他上面说的Light，而是Landscape，所以有些东西会不一样，有的没有有的多出来，你这里没有的不要随便加，尤其是widget那里，多出来的也可以暂时不用管。等你弄好大致结构装上新主题了在去慢慢改就好。
顺便说一句我觉得Light就挺好的，就是我现在用的这个。

有条件的还可以考虑Openshift等其他平台，因为Github毕竟是个程序员分享资源的地方，虽然page是可以搭博客不过最好还是以相关内容为主吧。
因此我这里也打算尽量放些计算机或程序相关的内容吧。也算是不白白占用人家资源。

Hexo搭建的是静态页面的博客，也就是没有各种交互的功能，但是如果纯粹只是写写东西发点图，贴个视频链接或是音乐的试听链接这种简单的功能还是都没问题的。
而评论功能也如你所见，我用的是Disqus（虽然国内被墙了，内地的比如“多说”也是不错的选择。当然，顶级域名和二级域名绑定也是可以实现的。

啰嗦了一大堆废话，这个Blog大概是我有生以来搞得最有技术含量的一个了（跪。希望它能一直存在并且走下去。
